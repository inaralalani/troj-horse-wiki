Software: REST API on Flask Python for tracking test validity and best practices in a clinical setting

What is it?

A software tool accessible through web application used to record infections, test results, and other peripheral factors.

Information that can be recorded for each patient:
  • Infection info: Chart info, duration, severity
  • Test results: Results of both traditional culture and rapid test
  • Genetic info: Species of bacteria causing infection, antibody sequence
  • Demographic info: Age, sex, race/ethnicity
  • Clinical practice info: Timing of test administration, treatments administered, clinical outcomes, qualitative charting info

Creates an easy-to-use database of information that can be analyzed to assess both the validity of our test, to inform best practices for the mitigation and treatment of PJIs, and associations between demographic factors and clinical outcomes.

User-friendliness and usefulness to other projects:

Though we are using it for a targeted purpose, designing this application in a modular fashion (i.e., no hard coding) and using Flask Python, a microframework suitable for beginners, other teams can easily adapt this application to collect data for their projects. REST APIs are also incredibly useful as they are modular and flexible, meaning that they can easily be adapted to handle multiple types of calls and return different data formats.
